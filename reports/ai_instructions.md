カレントディレクトリの `_pending.json` を Read で読んでください。これは現在止まっている/期限が迫っているタスクの一覧です。

その内容を本気で分析し、`ai-advice.json` を Write で書いてください。スキーマは厳密に次の通り:

{
  "id": "<_pending.json の id と完全一致させる>",
  "briefing": "<その日の戦略ブリーフィング。現在時刻(_pending.json の time)を踏まえ、優先順位とその理由を2〜4段落の日本語で。具体的に書く>",
  "tasks": [
    {
      "title": "<_pending.json の stuck_tasks[].title と完全一致>",
      "insight": "<そのタスク固有の見立てを1〜2文。一般論ではなく実データに即して>",
      "prompt": "<そのままAIに貼って続きを進められる、具体的で完成された質問文>"
    }
  ]
}

ルール:
- stuck_tasks の全件に対応する tasks 要素を必ず用意する。title は1文字も変えず完全一致させる(不一致だと統合されない)。
- id も _pending.json の id と完全一致させる。
- 分析の着眼点の例: 同一依頼者への放置=信頼リスク / 即勝ちできる軽量タスク / 本日締切で印刷リードタイムが必要 / 古すぎる案件は生死確認が先 / 全タスク0%なら進捗未記録の疑い / 現在時刻から逆算した今日の現実的な落とし所。
- 曖昧語(いい感じ/適度に/ある程度/だいたい/なんとなく)は使わない。固有名詞・数値・依頼者名で書く。
- ai-advice.json を書いたら完了。余計な説明やファイルは作らない。Bashやgitは実行しない。
